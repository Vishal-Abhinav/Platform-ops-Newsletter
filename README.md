## 🧭 What is This?

> *"We built the newsletter we wished existed when we were starting out."*

**Platform Ops** is a monthly technical newsletter covering the full **Platform Engineering & DevOps stack** — written by two practitioners who live in the trenches of Kubernetes, cloud infrastructure, and developer platforms.

After 2 years and 47 issues, we've gone **fully open source**. Every issue, every diagram, every word — free for the community forever.

## 🚀 Quick Start

### Clone & Explore

```bash
# Clone the repository
git clone https://github.com/vishal-abhinav/Tech_with_vishal_abhinav_Newsletter.git
cd Tech_with_vishal_abhinav_Newsletter

# Browse the latest issue
open 2026/03-March/issue-047/newsletter.html

# List all published newsletters
find . -name "newsletter.html" | sort
```

### Run Locally with a Dev Server

```bash
# Python (no install needed)
python3 -m http.server 8080
# → Open http://localhost:8080

# Node.js
npx serve .
# → Open http://localhost:3000
```

### Add a New Issue (Contributors)

```bash
# 1. Create a branch
git checkout -b issue/048-april-gateway-api

# 2. Scaffold the folder
mkdir -p 2026/04-April/issue-048/{diagrams,assets}
touch 2026/04-April/issue-048/newsletter.html
touch 2026/04-April/issue-048/SUMMARY.md

# 3. Commit and push
git add 2026/04-April/
git commit -m "📰 Issue #048: Gateway API — The Future of Ingress"
git push origin issue/048-april-gateway-api

# 4. Open a Pull Request on GitHub
```

## 🤝 Contributing

We welcome contributions from the community! Here's how:

### Ways to Contribute

```
🐛  Fix typos or broken links        → Open a PR directly
📝  Improve existing content         → Open a PR with changes
💡  Suggest a new topic              → Open a GitHub Issue
🖼️  Submit a diagram                 → Add to diagrams/ folder
🌍  Translate an issue               → Create a /translations folder
⭐  Star the repo                    → Takes 1 second, means a lot
```

### Contribution Guidelines

1. **Fork** the repository
2. **Branch** from `main`: `git checkout -b fix/your-description`
3. **Follow** the folder structure exactly
4. **Test** your HTML locally before submitting
5. **PR** with a clear title and description

> See [CONTRIBUTING.md](./CONTRIBUTING.md) for the full guide.
